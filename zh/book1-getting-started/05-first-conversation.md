# 第五章：第一次对话

## 一切从这里开始

你已经安装好 Claude Code 了。现在，让我们真正跑起来——不是读文档，是真实地使用它。

本章你会完成：
1. 启动 Claude Code
2. 问第一个问题："这个目录里有什么？"
3. 让它帮你创建一个 Hello World 网页
4. 理解"审批流程"是怎么回事
5. 学到写出好 prompt 的几个技巧

准备好了吗？我们开始。

---

## 启动 Claude Code

打开你的终端。

**第一步：进入一个文件夹**

Claude Code 需要在一个文件夹（目录）里工作。建议先创建一个专门用来练习的文件夹：

```bash
# 在桌面上创建一个新文件夹
mkdir ~/Desktop/claude-practice
# 进入这个文件夹
cd ~/Desktop/claude-practice
```

**第二步：启动 Claude Code**

```bash
claude
```

按下回车，你会看到欢迎界面出现。大致长这样：

```
Welcome to Claude Code!
Logged in as: 你的账号邮箱

Type your message below, or /help for available commands.

>
```

那个 `>` 符号就是等待你输入的光标。Claude Code 已经在运行了，就像打开了一扇门，里面有人在等着。

---

## 第一个 Prompt："这个目录里有什么文件？"

我们从最简单的开始。输入：

```
这个目录里有什么文件？
```

然后按回车。

**现在发生了什么：**

因为我们刚刚创建了一个空文件夹，Claude Code 会告诉你这个目录是空的。它可能会回复类似：

```
这个目录目前是空的，没有任何文件。

你想在这里开始一个新项目吗？我可以帮你创建文件或者初始化一个项目。
```

这验证了一件事：Claude Code 真的在看你的文件夹，而不是瞎说。

**如果你想试试有文件的目录，** 可以这样做：退出 Claude Code（输入 `exit` 或按 `Ctrl+C`），进入一个你已有的项目文件夹，再重新启动 `claude`，然后问同样的问题。你会看到它真实地列出文件结构，甚至能解释每个文件是干什么的。

---

## 进阶尝试："帮我创建一个 Hello World 网页"

这是第一个真正有意思的任务。在对话框里输入：

```
帮我创建一个 Hello World 网页，要好看一点，有蓝色背景和居中的大字。
```

按回车，然后观察接下来发生的事。

### 你会看到什么？

Claude Code 会先告诉你它打算做什么，然后询问你的许可。大致像这样：

```
我会创建一个 index.html 文件，包含：
- 蓝色渐变背景
- 居中显示的大字 "Hello World"
- 简单的样式

是否允许我创建这个文件？ [y/n]
```

输入 `y` 并按回车，Claude Code 就会创建文件。

完成后，它会说类似：

```
已创建 index.html。

你可以用浏览器打开它来查看效果：
  open index.html   （Mac）
  start index.html  （Windows）

需要我修改任何样式吗？
```

**在终端里打开文件：**

```bash
open index.html       # Mac
start index.html      # Windows
xdg-open index.html   # Linux
```

你的浏览器会打开，显示你刚才"创建"的网页——但实际上是 Claude Code 帮你写的。

---

## 理解发生了什么：Claude Code 使用了工具

在你看不见的地方，Claude Code 刚才做了这些事：

1. **理解你的需求**：把你的自然语言描述（"好看、蓝色背景、居中大字"）转化成具体的技术实现方案
2. **使用"写入文件"工具**：创建了 `index.html` 文件，写入了完整的 HTML 和 CSS 代码
3. **告诉你结果**：报告它做了什么，并给出后续建议

这就是我们在第三章讲的"工具系统"在实际中的样子。你用中文说了一句话，它背后调用了真实的操作。

---

## 审批流程：当 Claude Code 征求许可时

你可能注意到，Claude Code 在创建文件前问了你"是否允许"。这是它的**权限系统**在起作用（第三章介绍过的）。

让我们来理解不同类型的提示：

### 类型一：文件修改确认

当 Claude Code 要创建或修改文件时：

```
我将要创建以下文件：
  📄 index.html (新文件，约 45 行)

是否继续？ [y/n]
```

**你应该怎么做：**
- 看清楚它要创建/修改哪个文件
- 如果文件名和你预期的一致，输入 `y`
- 如果不对，输入 `n`，然后告诉它哪里不对

### 类型二：命令执行确认

当 Claude Code 要运行命令时：

```
我将要运行以下命令：
  $ npm install

这个命令会安装 Node.js 依赖包。是否允许？ [y/n]
```

**你应该怎么做：**
- 如果你不理解这个命令，可以直接问："这个命令是干什么的？有什么风险吗？"
- 理解后再决定是否允许

### 黄金原则

**不理解就不要按 `y`**。Claude Code 的设计就是为了让你可以在每一步做决定。它不会因为你多问一个问题而生气。

---

## 继续对话：迭代和调整

假设你对刚才创建的网页不太满意，想改改。直接说：

```
把背景改成深紫色，字体改大一点，加上一个"开始"按钮。
```

Claude Code 会：
1. 读取现有的 `index.html` 文件
2. 规划修改方案
3. 告诉你它打算改哪些地方
4. 征求你的确认
5. 修改文件

这就是迭代的样子——你不需要每次都从零开始，可以在现有基础上不断调整。

## 写好 Prompt 的技巧

经过这一章的实践，我们来总结几个让 Claude Code 更好地理解你的技巧。

### 技巧一：说清楚你想要的结果，而不只是方法

**不够好的 prompt：**
```
写一个 HTML 文件
```

**更好的 prompt：**
```
帮我创建一个个人简历网页，包含姓名"张明"、职位"产品经理"、
三段工作经历，和联系方式。风格简洁专业，黑白配色。
```

说清楚"是什么"和"为了什么"，Claude Code 做出来的结果会更接近你的期待。

### 技巧二：提供背景信息

```
这是一个用于向客户展示的演示网站，受众是非技术人员，
所以界面要尽量简单直观。帮我在首页加一个"开始使用"的按钮……
```

背景信息帮助 Claude Code 做出更合适的判断。

### 技巧三：如果第一次不对，直接描述哪里不对

```
按钮颜色太浅了，在白色背景上看不清楚。改成深蓝色 (#1a56db)。
```

你不需要重写整个需求。直接说哪里不对就行——这就是对话式工作的优势。

### 技巧四：分步骤而不是一次性说太多

不要：
```
帮我创建一个完整的电商网站，有商品列表、购物车、用户登录、
支付系统、后台管理……
```

要：
```
我们来一步步创建一个简单的商品展示页面。
第一步：先帮我创建商品列表页，展示 6 个示例商品，每个有图片、名称和价格。
```

分步走，每步确认，比一次性丢一个大任务要稳健得多。

### 技巧五：让 Claude Code 先"看"，再"做"

在做大的改动前，先让它告诉你它看到了什么：

```
先帮我看看这个项目的结构，告诉我主要文件有哪些，
然后告诉我如果要添加用户登录功能，应该从哪里入手。
```

先规划，再动手，减少返工。

## 新手常见错误

### 错误一：直接按 `y` 而不看提示

Claude Code 征求确认时，有些人习惯性地直接按 `y`。养成先读一下提示的习惯——就几秒钟，但能防止很多意外。

### 错误二：prompt 太模糊

"帮我做得好看一点"——"好看"对不同人来说完全不同。说清楚你心目中的"好看"：颜色、风格、参考案例、目标用户。

### 错误三：一次性提太多要求

任务越具体越好做，任务越大越容易出偏差。把大任务分解成小步骤，一步一步来。

### 错误四：遇到问题就放弃，而不是继续对话

Claude Code 第一次做的不对，不要退出重来。直接在同一个对话里说"不对，问题在于……"——它会根据你的反馈调整，而且有上下文，调整起来会更准确。

### 错误五：忘记它是在你的真实文件上操作

这不是沙盒游戏，它修改的是你电脑上真实的文件。重要的项目在操作前先备份，或者确保你在用 Git 做版本控制。

## 常用命令备忘

在 Claude Code 的对话界面里，你可以用这些特殊命令：

| 命令 | 作用 |
|------|------|
| `/help` | 显示所有可用命令 |
| `/clear` | 清除当前对话历史 |
| `/model` | 切换使用的 AI 模型 |
| `exit` 或 `Ctrl+C` | 退出 Claude Code |
| `Shift+Tab` | 切换权限模式（默认/自动接受/计划模式） |
| 按 `?` | 查看快捷键列表 |
| 按 `↑` | 查看历史命令 |
