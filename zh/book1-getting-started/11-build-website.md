# 第十一章：从零搭一个网站 — 实战教程

## 开始之前

本章是一个完整的实战教程。我们要做什么？

**目标：** 用 Claude Code 从零搭建一个个人作品集网站，最终部署到 GitHub Pages，让全世界都能访问。

**你需要的：**
- 已安装 Claude Code（参考第四章）
- 已安装 Git 和 GitHub 账号
- 约 30-60 分钟时间

**你不需要的：**
- 任何前端开发经验
- 了解 HTML/CSS/JavaScript

我们将用完整的对话展示每个步骤，你可以直接跟着做。

---

## 第一步：创建项目文件夹，启动 Claude Code

打开终端，运行：

```bash
mkdir my-portfolio
cd my-portfolio
claude
```

---

## 第二步：告诉 Claude 你想做什么

```
你好！我想搭建一个个人作品集网站，用来展示我自己。
我完全不懂前端开发，需要你帮我从零开始。

网站需要包含：
1. 首页 — 自我介绍（姓名：李明，职业：产品经理）
2. 关于我 — 更详细的介绍和技能
3. 项目展示 — 展示 3 个我做过的项目
4. 联系方式 — 邮箱和社交媒体链接

风格：简洁、现代、专业，以蓝白色为主。

请先告诉我整体计划，然后一步步来。
```

**Claude 的回应大致如下：**

```
好的！我来帮你搭建这个个人作品集网站。

【整体计划】
我们将创建一个纯 HTML/CSS 网站，不需要后端，可以
直接部署到 GitHub Pages。

计划创建以下文件：
- index.html     ← 首页（自我介绍）
- about.html     ← 关于我
- projects.html  ← 项目展示
- contact.html   ← 联系方式
- style.css      ← 统一的样式文件

先从 index.html 和 style.css 开始，然后
逐步添加其他页面。

我们从创建 index.html 开始吗？
```

## 第三步：创建首页

```
好的，开始吧，先创建首页。
```

**Claude 会创建 `index.html`，然后展示 diff 让你确认。确认后，文件就建好了。**

你可以在终端预览：

```bash
open index.html
```

或者在对话里说：

```
帮我在浏览器里打开 index.html 看效果
```

**第一版效果看起来有点简单？继续迭代：**

```
首页看起来太素了。帮我：
1. 加一张头像占位图（先用一个彩色圆形代替）
2. 把名字字体做大一些，更醒目
3. 加一个"了解我"按钮，点击后跳转到 about.html
```

Claude 会修改文件，你再次刷新浏览器就能看到变化。

---

## 第四步：添加样式

```
帮我创建 style.css 文件，定义整个网站的统一风格：
- 主色：#1a56db（深蓝色）
- 辅助色：#f9fafb（浅灰色背景）
- 字体：系统默认字体，标题加粗
- 导航栏：固定在顶部，有网站名称和页面链接
- 确保所有四个页面都引入这个 CSS 文件
```

---

## 第五步：添加其余页面

按照同样的方式，一次一个页面：

```
帮我创建 about.html——"关于我"页面。
内容包括：
- 简短的个人故事（用虚构内容占位就好，我之后自己改）
- 技能标签：产品设计、用户研究、数据分析、项目管理
- 工作经历时间线（3 条虚构记录）
```

```
帮我创建 projects.html——项目展示页面。
展示 3 个项目，每个项目包括：
- 项目名称
- 一句话描述
- 技术/工具标签
- "查看详情"按钮（先不用有实际链接）

使用卡片布局，三列排列，每列一个项目。
```

```
帮我创建 contact.html——联系方式页面。
包含：
- 一个联系表单（姓名、邮箱、消息三个字段）
- 社交媒体链接图标（GitHub、LinkedIn、微信）
- 我的邮箱地址：example@email.com
```

---

## 第六步：做响应式设计

响应式设计让网站在手机上也好看。

```
帮我给网站添加响应式支持。
要求：
- 手机屏幕上，导航菜单变成汉堡菜单（点击展开）
- 项目展示页在手机上变成单列
- 所有文字在小屏幕上保持可读大小（至少 16px）
- 按钮和链接要足够大，方便手指点击
```

---

## 第七步：最终优化

```
帮我做几个最后的优化：
1. 所有页面添加网站标题标签（<title>），格式为"李明 - 产品经理"
2. 给页面添加 meta description，用于搜索引擎
3. 确保所有页面之间的导航链接都是正确的
4. 检查有没有 HTML 语法错误
```

## 第八步：部署到 GitHub Pages

当你对网站满意后，可以把它部署到 GitHub Pages，让所有人都能访问。

### 8.1 初始化 Git 仓库

```
帮我把这个项目初始化为 Git 仓库，然后做第一次提交。
```

Claude 会运行：
```bash
git init
git add .
git commit -m "feat: initial portfolio website"
```

### 8.2 在 GitHub 创建仓库

如果你安装了 GitHub CLI（gh），可以让 Claude 帮你：

```
帮我在 GitHub 上创建一个新仓库，叫 my-portfolio，
然后把代码推送上去。
```

Claude 会运行：
```bash
gh repo create my-portfolio --public
git remote add origin https://github.com/你的用户名/my-portfolio.git
git push -u origin main
```

如果没有安装 gh，去 GitHub.com 手动创建仓库，然后：

```
帮我把代码推送到 GitHub，仓库地址是：
https://github.com/你的用户名/my-portfolio.git
```

### 8.3 开启 GitHub Pages

1. 打开你的 GitHub 仓库页面
2. 点击 Settings（设置）
3. 左侧菜单找到 Pages
4. Source 选择：Deploy from a branch
5. Branch 选择：main，文件夹选择：/(root)
6. 点击 Save

等 1-2 分钟，你的网站就会在以下地址上线：

```
https://你的用户名.github.io/my-portfolio/
```

```
帮我给网站加一个暗色主题切换按钮，

```

```
帮我检查一下网站的加载速度，看有没有可以优化的地方。
```

## 完整对话示意

这是这个教程的对话流程概览：

```
你 → "帮我做个作品集网站，计划一下"
Claude → 列出计划

你 → "好，开始"
Claude → 创建 index.html + style.css

你 → "首页太素，改一下" / "加个头像" / "按钮颜色深一点"
Claude → 迭代修改

你 → "帮我做 about.html"
Claude → 创建页面

你 → "做 projects.html"
Claude → 创建卡片布局

你 → "做 contact.html"
Claude → 创建联系表单

你 → "加响应式"
Claude → 添加移动端适配

你 → "帮我提交 Git，部署到 GitHub Pages"
Claude → 提交并引导部署
```

整个过程大约 30-60 分钟，最终你拥有一个真实在线的个人网站。

## 小结

这一章展示了 Claude Code 的完整工作流：

- **从零开始** 不需要任何模板或脚手架
- **迭代改进** 每次说哪里不满意，Claude 就改哪里
- **多文件协调** Claude 自动维护所有页面之间的一致性
- **从本地到线上** 包含完整的 Git 和部署流程

你在本章用到的所有技能——提需求、审改动、迭代调整、Git 提交、部署——在所有项目中都是通用的。
