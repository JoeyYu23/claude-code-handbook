# 第三章：工作原理 — 建立心智模型

## 为什么要了解"工作原理"？

你不需要知道汽车发动机的每一个零件才能开车，但了解"踩油门车会加速、踩刹车会减速"这样的基本原理，会让你成为更好的司机。

Claude Code 也是一样。你不需要懂 AI 技术，但理解它的基本工作方式，会让你知道：
- 为什么它有时候需要几分钟才能完成任务
- 为什么它在做某些操作前会停下来问你
- 为什么有时候它会"迷路"，而你该怎么把它拉回来

---

## 核心模型：对话 + 行动的循环

Claude Code 的工作方式可以用三个词概括：**理解、行动、验证**。

当你给它一个任务，它会进入这样的循环：

```
你说话
  ↓
Claude 理解你的意图
  ↓
Claude 使用工具收集信息（读文件、搜索、运行命令）
  ↓
Claude 采取行动（修改文件、创建文件、运行程序）
  ↓
Claude 验证结果（看看有没有错误）
  ↓
Claude 告诉你结果
  ↓
你说话（继续、调整、确认）
  ↓
（循环往复）
```

> **类比：** 想象你雇了一个装修师傅来翻新你的厨房。你告诉他想要什么效果，他会先量尺寸（收集信息），再动手改（行动），最后叫你来看看（验证），然后问你"这样可以吗？"如果你说"窗户往左移一点"，他会继续下一轮。Claude Code 就是这种来回循环的工作模式。

---

## 工具系统：Claude Code 的"手"

Claude Code 的语言模型本身只能产生文字。让它变得"会干活"的，是一套**工具系统**——这些工具就是它伸出去操作你电脑的"手"。

官方文档把这些工具分成五大类。我们用生活化的语言来解释每一类：

---

### 工具一：文件操作（读、写、改、建）

**它能做什么：** 读取文件内容、创建新文件、修改已有文件、重命名文件、重新组织文件夹结构。

**类比：** 就像一个助手可以帮你翻阅文件柜里的资料、帮你打印新文件、帮你在已有文件上涂改、帮你把文件分类整理。

**实际例子：**
- 你说："帮我看看 `app.py` 里有什么"→ Claude Code 读取那个文件的内容
- 你说："帮我新建一个 `index.html`"→ Claude Code 创建一个新文件，写入内容
- 你说："把登录函数改成支持谷歌登录"→ Claude Code 找到那个函数，修改它

---

### 工具二：搜索（在你的项目里找东西）

**它能做什么：** 按文件名查找文件，用关键词在所有文件里搜索，用更复杂的规则（正则表达式）匹配内容。

**类比：** 就像电脑上的"全局搜索"功能（Mac 上的 Spotlight，Windows 上的文件搜索），但 Claude Code 不只找到文件，还能理解找到的内容。

**实际例子：**
- 你说："登录相关的代码在哪里？"→ Claude Code 搜索所有文件，找到包含"login"、"auth"等关键词的文件
- 你说："这个项目里有没有用到数据库的地方？"→ Claude Code 扫描所有文件，汇报它的发现

---

### 工具三：执行（运行命令）

**它能做什么：** 在你的终端里运行任何命令——安装软件包、启动服务器、运行测试、使用 Git、执行脚本。

**类比：** 就像一个助手可以帮你在键盘上打命令、按回车执行。不同的是，它能看到执行的结果，并根据结果决定下一步。

**实际例子：**
- 你说："帮我运行一下测试，看看有没有报错"→ Claude Code 运行 `npm test` 或 `pytest`，读取输出，告诉你哪里失败了
- 你说："帮我把这次改动提交到 Git"→ Claude Code 运行 `git add`、`git commit`，写好提交信息

> **重要：** 执行命令是有影响的操作，Claude Code 在运行命令前会征求你的同意（默认模式下）。

---

### 工具四：网页搜索（上网查资料）

**它能做什么：** 搜索网页、获取在线文档、查找错误信息的解决方案。

**类比：** 就像你在工作时旁边有一个可以帮你 Google 问题的助手，而且他能立刻把找到的信息用到你的项目上，而不是把链接甩给你让你自己去看。

**实际例子：**
- 你说："这个报错信息是什么意思？怎么解决？"→ Claude Code 搜索这个错误，找到解决方案，直接帮你修改
- 你说："帮我查一下 Python 的最新版是多少"→ Claude Code 搜索后告诉你

---

### 工具五：代码智能（理解代码关系）

**它能做什么：** 查看类型错误和代码警告、跳转到函数定义、找到哪些地方调用了某个函数。

**类比：** 就像一个读过整本书的读者，能告诉你"第 42 章提到的那个角色，在第 8 章第一次出现"——Claude Code 能理解代码里各部分之间的关系。

---

## 上下文窗口（Context Window）：有限大小的白板

这是理解 Claude Code 行为的关键概念之一。

**什么是上下文窗口？**

想象 Claude Code 工作时，有一块大白板。所有它当前"知道"的信息都写在这块白板上：
- 你们的对话历史
- 它读取的文件内容
- 运行命令的输出结果
- 你在 CLAUDE.md 里写的指令

这块白板**有大小限制**。当信息填满了，旧的内容会被擦掉或者压缩，为新信息腾出空间。

**这意味着什么？**

- 一个很长的对话后，Claude Code 可能会"忘记"早些时候你说过的事情
- 如果你打开了很多文件，白板空间会更快被用完
- 你的持久性指令应该写在 `CLAUDE.md` 文件里，而不是每次对话都重复说

> **类比：** 就像你工作的桌子面积有限。桌子上能放的东西多，你工作效率就高；但如果堆满了旧资料，你就得清理出空间来处理新任务。

**实践建议：**
- 如果你发现 Claude Code 开始"忘事"，可以开一个新的对话（会话）
- 把重要的规则和偏好写进 `CLAUDE.md`，这些文件每次会话开始都会被读取

## 权限系统：Claude 做危险操作前会问你

这是 Claude Code 设计里一个很重要的安全机制。

### 为什么需要权限系统？

有些操作是可逆的（比如读取文件——读了不改，没有影响），有些操作是不可逆的或者有重大影响的（比如删除文件、在数据库里插入数据、发送邮件）。Claude Code 对"有影响的操作"会先停下来，问你"我打算这么做，可以吗？"

### 三种权限模式

Claude Code 有三种工作模式，你可以用 `Shift+Tab` 键在它们之间切换：

**默认模式（Default）：**
每次修改文件或运行命令前，Claude 都会告诉你它打算做什么，等你确认。这是最安全的模式，适合新手。

**自动接受编辑模式（Auto-accept edits）：**
Claude 可以直接修改文件，不需要每次都问你，但运行命令还是会确认。当你信任 Claude 的判断、不想一直点"确认"时可以用。

**计划模式（Plan mode）：**
Claude 只会读取文件和搜索，不会做任何修改。它会制定一个计划给你看，等你批准后才开始行动。适合在正式动手前先思考方案。

### 确认提示长什么样？

当 Claude Code 要运行一个命令时，你会看到类似这样的提示：

```
Claude 想要运行以下命令：
  npm install express

是否允许？ [y/n/说明原因]
```

你可以：
- 输入 `y` 允许
- 输入 `n` 拒绝
- 输入你的疑虑，Claude 会解释为什么需要这个命令

**重要原则：如果你不理解 Claude 要做的操作，永远可以说"先解释一下这个命令是干什么的"。**

## 整个工作流程：一次完整的对话

让我们用文字走一遍完整的流程，帮你建立直觉。

**场景：你让 Claude Code 帮你修复一个报错**

```
你：         "程序运行时报错 'Cannot find module express'，帮我修复。"

Claude Code  思考：这可能是缺少依赖包的问题。我需要先确认一下。

Claude Code  [使用文件工具] 读取 package.json，看看项目依赖列表

Claude Code  [使用文件工具] 读取 package-lock.json 或 node_modules 目录

Claude Code  发现：express 包确实没有安装

Claude Code  告诉你："发现问题了，express 包没有安装。
             我打算运行 'npm install express' 来安装它，可以吗？"

你：         "可以"

Claude Code  [使用执行工具] 运行 npm install express

Claude Code  [查看命令输出] 安装成功

Claude Code  告诉你："安装完成！现在重新运行程序试试。"

你：         "还是报错，这次错误是 'listen EADDRINUSE: address already in use'"

Claude Code  思考：这是端口被占用的错误……
             [继续下一轮循环]
```

注意这个过程：Claude Code 不是一次性给你答案，而是一步步探索、行动、反馈。这就是"智能体循环"（agentic loop）的样子。

---

## 小结：你需要记住的三件事

1. **Claude Code 通过工具"动手"**——读文件、改文件、跑命令、搜网页。每次动手前，它会先"思考"。

2. **上下文窗口是有限的白板**——很长的对话后它可能"忘事"，重要规则写进 `CLAUDE.md`。

3. **权限系统是你的安全网**——它在做有影响力的操作前会问你，你永远有权说不。

理解了这些，你就有了一个可靠的心智模型。接下来，让我们真正动手——安装 Claude Code。
