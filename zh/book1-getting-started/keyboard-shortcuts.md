# 附录 C：快捷键

本附录收录 Claude Code 在各个使用环境中的快捷键，方便随时查阅。

---

## 终端 CLI 快捷键

这些快捷键在终端里运行 `claude` 时有效。

### 基本操作

| 快捷键 | 功能 |
|--------|------|
| `Enter` | 发送消息 |
| `Shift+Tab` | 切换权限模式（默认 → 自动接受 → 计划模式，循环） |
| `Ctrl+C` | 中断当前操作 / 退出 Claude Code |
| `Ctrl+D` | 退出 Claude Code（EOF 信号） |
| `Ctrl+L` | 清屏（不清除对话历史） |

### 输入编辑

| 快捷键 | 功能 |
|--------|------|
| `↑` / `↓` 方向键 | 浏览历史输入 |
| `Ctrl+A` | 光标跳到行首 |
| `Ctrl+E` | 光标跳到行尾 |
| `Ctrl+K` | 删除光标到行尾的内容 |
| `Ctrl+U` | 删除光标到行首的内容 |
| `Ctrl+W` | 删除光标前的一个单词 |
| `Tab` | 自动补全命令或文件名 |

### 多行输入

| 快捷键 | 功能 |
|--------|------|
| `\` + `Enter` | 在行末加反斜杠后回车，继续输入下一行 |
| `Option+Enter`（Mac） | 直接换行（不发送） |

### 快速执行命令

| 输入方式 | 功能 |
|----------|------|
| `!命令` | 直接在 Claude 对话框里运行 shell 命令（如 `!git status`） |

## Slash 命令（通用）

在 Claude Code 对话框里输入以下命令，在终端和 VS Code 插件中均可使用（部分命令仅限终端）。

### 常用命令

| 命令 | 功能 |
|------|------|
| `/help` | 显示帮助，列出所有可用命令 |
| `/clear` | 清除当前对话历史，开始新对话 |
| `/compact` | 压缩对话上下文（当对话很长时使用） |
| `/memory` | 查看和管理 CLAUDE.md 及 Auto Memory 文件 |
| `/model` | 切换使用的 AI 模型 |
| `/config` | 打开设置界面 |

### 项目和会话

| 命令 | 功能 |
|------|------|
| `/init` | 自动分析项目并生成 CLAUDE.md 初稿 |
| `/resume` | 恢复之前的对话 |
| `/ide` | 连接到当前打开的 VS Code |
| `/desktop` | 连接到 Claude Desktop 应用 |

### 权限和安全

| 命令 | 功能 |
|------|------|
| `/permissions` | 查看和管理当前的权限设置 |

### 查看信息

| 命令 | 功能 |
|------|------|
| `/status` | 显示当前 session 状态（token 使用量等） |
| `/usage` | 显示本次对话的 token 使用情况 |
| `?` | 在等待输入时按 `?`，显示快捷键提示 |

## VS Code 插件快捷键

这些快捷键在 VS Code 里安装了 Claude Code 插件后可用。

### 打开和导航

| 快捷键（Mac） | 快捷键（Windows/Linux） | 功能 |
|--------------|------------------------|------|
| `Cmd+Shift+X` | `Ctrl+Shift+X` | 打开扩展市场 |
| `Cmd+Shift+P` | `Ctrl+Shift+P` | 命令面板（输入 "Claude Code" 查找相关命令） |
| `Cmd+Esc` | `Ctrl+Esc` | 在编辑器和 Claude 对话框之间切换焦点 |
| `Cmd+Shift+Esc` | `Ctrl+Shift+Esc` | 在新标签页打开 Claude 对话 |
| 点击状态栏 "✱ Claude Code" | 同左 | 打开 Claude Code 面板（最稳定的入口） |

### 对话操作

| 快捷键（Mac） | 快捷键（Windows/Linux） | 功能 |
|--------------|------------------------|------|
| `Cmd+N` | `Ctrl+N` | 新建对话（需要 Claude 对话框聚焦时） |
| `Enter` | `Enter` | 发送消息 |
| `Shift+Enter` | `Shift+Enter` | 在消息框内换行（不发送） |

### 代码引用

| 快捷键（Mac） | 快捷键（Windows/Linux） | 功能 |
|--------------|------------------------|------|
| `Option+K` | `Alt+K` | 插入当前文件和选中行的 @-mention 引用 |

使用方法：在编辑器里选中代码，然后按 `Option+K`（Mac）/ `Alt+K`（Windows），Claude 对话框里会自动插入 `@文件名#起始行-结束行` 的引用。

## 权限模式切换（快速参考）

| 操作 | 方法 |
|------|------|
| 在模式间循环切换 | `Shift+Tab`（终端）或点击对话框底部模式指示器（VS Code） |
| 查看当前模式 | 对话框底部显示当前模式 |
| 临时设置默认模式 | VS Code 设置：`claudeCode.initialPermissionMode` |

三种模式快速提示：
- **默认模式**：每步操作都问你
- **自动接受模式**：文件编辑不问，命令执行还问
- **计划模式**：先列计划，你批准后再动手

---

## 确认提示快速回应

当 Claude 需要你确认操作时：

| 按键 | 含义 |
|------|------|
| `y` 或 `Y` | 允许（Yes） |
| `n` 或 `N` | 拒绝（No） |
| `Enter`（部分情况） | 接受默认选项（通常是 Yes） |

## 实用输入技巧

### 一次性指令（非交互模式）

在终端里不进入交互界面，直接执行单次任务：

```bash
claude -p "帮我检查 src/app.js 有没有语法错误"
```

### 通过管道传入内容（终端专属）

```bash
# 把错误日志传给 Claude 分析
cat error.log | claude -p "分析这些错误，找出最常见的问题"

# 让 Claude 审查 git diff
git diff HEAD~1 | claude -p "这些改动有什么问题吗？"

# 批量处理文件
ls src/*.js | claude -p "列出这些文件里哪些可能有导出问题"
```
